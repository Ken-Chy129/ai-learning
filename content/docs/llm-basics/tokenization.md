# Tokenization 分词

Tokenization 是将文本拆分为模型可以处理的基本单元（token）的过程。

## 常见分词方法

### BPE（Byte Pair Encoding）

GPT 系列使用的分词方法。核心思想是迭代合并出现频率最高的字符对。

**示例：**

```
原始文本: "low lower lowest"
初始: ['l', 'o', 'w', ' ', 'l', 'o', 'w', 'e', 'r', ...]
合并 'l'+'o' → 'lo': ['lo', 'w', ' ', 'lo', 'w', 'e', 'r', ...]
合并 'lo'+'w' → 'low': ['low', ' ', 'low', 'e', 'r', ...]
```

### WordPiece

BERT 使用的分词方法，与 BPE 类似，但选择合并的依据是最大化语言模型的似然。

### SentencePiece

Google 开发的语言无关分词工具，直接在原始文本上操作，不依赖预分词。LLaMA 和许多多语言模型使用此方法。

## 词汇表大小的影响

| 词汇表大小 | 优点 | 缺点 |
|-----------|------|------|
| 较小（~30K） | 模型参数少 | 序列更长，推理慢 |
| 较大（~100K） | 序列更短，推理快 | 嵌入层参数多 |

## 特殊 Token

常见的特殊 token：

- `<BOS>` / `<s>` — 序列开始
- `<EOS>` / `</s>` — 序列结束
- `<PAD>` — 填充
- `<UNK>` — 未知词
