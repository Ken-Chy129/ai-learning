# 常用提示技巧

## Zero-Shot Prompting

直接给模型下达任务指令，不提供示例。

```
将以下句子翻译成英文：今天天气真好。
```

## Few-Shot Prompting

在提示中提供少量示例，引导模型理解任务格式。

```
将中文翻译成英文：

中文：猫坐在垫子上。
英文：The cat sat on the mat.

中文：今天天气真好。
英文：
```

## Chain-of-Thought（思维链）

引导模型逐步推理，提升复杂任务的准确性。

```
Q: 一个商店有 23 个苹果，卖掉了 17 个，又进货了 12 个，现在有多少个？
A: 让我一步步思考：
1. 开始有 23 个苹果
2. 卖掉 17 个：23 - 17 = 6 个
3. 又进货 12 个：6 + 12 = 18 个
答案：18 个苹果
```

## ReAct（Reasoning + Acting）

结合推理和行动，让模型能调用外部工具完成任务。

```
问题：2024年奥运会在哪里举行？
思考：我需要查询2024年奥运会的举办地。
行动：搜索("2024年奥运会举办城市")
观察：2024年夏季奥运会在法国巴黎举行。
答案：2024年奥运会在法国巴黎举行。
```

## 提示优化原则

1. **明确具体** — 清楚描述你想要的输出格式和内容
2. **提供上下文** — 给模型足够的背景信息
3. **分步指令** — 将复杂任务拆分成多个步骤
4. **设定角色** — 让模型扮演特定角色可以提升输出质量
5. **限定范围** — 明确告诉模型不应该做什么
